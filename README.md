# medical-charges-prediction

医療費（Medical Charges）データを対象に、データ探索（EDA）から仮説検証型の特徴量エンジニアリング、そして複数の機械学習アルゴリズム（線形モデル、正則化モデル、決定木系アンサンブルモデル）の最適化までを一気通貫で実施したリポジトリです。
本プロジェクトでは、単に高度なモデルを適用するだけでなく、「ドメイン知識に基づいたシグナルの切り出し」と「アルゴリズムの特性の不整合の解消」を数理的・統計的根拠に基づいて検証しています。

---

## 1. 特徴量エンジニアリングの意図と背景

データ探索の段階において、医療費の分布に線形回帰では捉えきれない非線形な不連続性（特定の条件下での急激な跳ね上がり）が確認されました。具体的には、「喫煙者（`smoker_yes == 1`）」かつ「BMI30以上（`bmi >= 30`：医学的な肥満基準）」のセグメントにおいて、医療費がマクロ的に乖離して高額化する統計的傾向を発見しました。

このシグナルをモデルに直接教え込むため、以下のバイナリフラグを新しく定義しました。

```python
df_features['high_bmi_smoker'] = ((df_features['bmi'] >= 30) & (df_features['smoker_yes'] == 1)).astype(int)
```

### 意図的な役割分担
BMI30という閾値は医学的基準に基づいた大雑把な区切りであり、これだけでは「BMI35」や「BMI45」といったグループ内のグラデーション（線形・非線形な変化）を表現しきれません。
しかし、本プロジェクトでは後続のフェーズで決定木ベースのアルゴリズム（Random ForestやLightGBM）を検証に組み込むことを見越していました。データ加工段階ではあえて「塊を特定するフラグ」の作成に留め、グループ内の微細な分岐構造の学習は決定木モデルの表現力に委ねるという、**「特徴量エンジニアリングとモデルの役割分担」**を意図して設計しています。

---

## 2. 検証プロセスおよびモデル実装

モデルの実装においては、最も単純で解釈性の高い線形回帰をベースライン（基準スコア）として設定し、段階的に複雑なモデルへとステップアップしながら汎化性能を検証しました。また、ハイパーパラメータの探索にはベイズ最適化アルゴリズムである **Optuna** を採用し、5分割交差検証（5-Fold Cross Validation）を用いて過学習を厳格に抑制しました。

### 検証対象モデル
1. **Linear Regression**: ベースラインの構築。
2. **Ridge / Lasso / ElasticNet**: 多重共線性の懸念に対する正則化（L1/L2ペナルティ）の影響を検証。
3. **Random Forest (Optuna最適化)**: バギングによる非線形関係のキャプチャと、パラメータ空間（100回試行）の探索。
4. **LightGBM (Optuna最適化)**: 現代のテーブルデータ分析において業界標準である勾配ブースティングによる限界精度の追求（100回試行）。

---

## 3. 検証結果（モデル性能比較）

すべてのモデルについて、学習データ内での交差検証を経て、完全に未知のデータであるテストデータ（全体の20%）に対する予測精度を評価しました。

| モデル | ハイパーパラメータ（最適解） | RMSE (ドル) | R2 Score (決定係数) |
| :--- | :--- | :--- | :--- |
| **Linear Regression** | デフォルト | 4,458 | 0.8720 |
| **Ridge** | `alpha`: 1.0 | 4,455 | 0.8722 |
| **Lasso** | `alpha`: 1.0 | 4,458 | 0.8720 |
| **ElasticNet** | `alpha`: 0.01, `l1_ratio`: 0.8 | 4,452 | 0.8723 |
| **Random Forest** | `n_estimators`: 200, `min_samples_split`: 10, ... | 4,486 | 0.8703 |
| **LightGBM** | `max_depth`: 3, `num_leaves`: 17, `min_child_samples`: 30, ... | **4,397** | **0.8754** |

---

## 4. 統計的・数理的考察

### 4.1. 特徴量重要度（Gain基準）の解析
最終的に最高精度を達成したLightGBMにおける特徴量重要度（ゲイン基準：損失の減少量への寄与度）は以下の結果となりました。

```text
         Feature   Importance
 high_bmi_smoker 3.414480e+12
      smoker_yes 5.997789e+11
             age 5.417693e+11
             bmi 4.689685e+10
        children 2.848160e+10
region_southwest 4.824830e+09
        sex_male 1.329253e+09
region_southeast 5.226702e+08
region_northwest 0.000000e+00
```

この結果から、独自に作成した `high_bmi_smoker` が、2位の `smoker_yes` や3位の `age` に圧倒的な大差をつけて予測に寄与していることが数理的に証明されました。これは、単に「喫煙」「BMI」を個別に扱うだけでは捉えきれなかった**複合的なリスク**を、作成したフラグが的確に表現できていることを示しています。

### 4.2. なぜLightGBMが最高精度を達成したのか
線形モデル（ElasticNet: RMSE 4,452）やランダムフォレスト（RMSE 4,486）を抑え、LightGBMが最高精度（RMSE 4,397）に達した背景には、Optunaによって選択されたパラメータ構造に明確な理由があります。

* **過学習の徹底的な抑制**
  選出されたパラメータは `max_depth: 3`（極めて浅い木）、`num_leaves: 17`、`min_child_samples: 30`（1つの葉に最低30件のデータを要求）という、強力なブレーキがかけられた構造でした。
* **構造の平滑化**
  このデータセットは総サンプル数が約1,300件と機械学習としては比較的少規模です。ここに複雑で深い決定木を適用すると、ノイズを過剰に学習して汎化性能が低下します（Random Forestの精度が100回試行でも飽和・微減した原因はここにあると考えられます）。
  LightGBMは、浅い木（弱学習器）を段階的に構築し、前段の誤差を少しずつ修正していく勾配ブースティングの特性を持ちます。これが今回の少規模データにおける過学習の抑制と非線形シグナルの回収のバランスに合致したため、テストデータに対して最も頑健なモデルとなりました。

### 4.3. モデルによる不要なノイズの自動排除
重要度において `region_northwest` が `0.000000e+00`（完全なゼロ）となっています。
これは、LightGBMが木を成長させる全プロセスにおいて、該当の地域情報が損失を減らすために1度も役に立たなかったと判断したことを意味します。正則化モデル（Lasso）が係数をゼロにするアプローチと同様のノイズ排除を、LightGBMが構造的に自動実行した形であり、医療費予測において特定の地域差は考慮に値しないノイズであるというドメイン的な知見を数学的に裏付けています。

---

## 5. 結論
本プロジェクトは、高度なアルゴリズムの選定（LightGBM + Optuna）のみに依存するのではなく、データの歪みを捉えた仮説主導の特徴量エンジニアリング（`high_bmi_smoker`）を前段に挟むことで、モデルの表現力を上昇させることができると実証しました。
